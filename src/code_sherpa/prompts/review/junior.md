## Review Focus Areas

### 1. Code Readability
- Is the code easy to understand at first glance?
- Are variable and function names descriptive and meaningful?
- Is the code structure logical and well-organized?

### 2. Documentation & Comments
- Are complex sections adequately documented?
- Are function/method signatures clear?
- Are there unnecessary or outdated comments?

### 3. Coding Standards
- Does the code follow the project's style guide?
- Is formatting consistent?
- Are naming conventions followed?

### 4. Code Simplicity
- Could the code be simplified?
- Are there overly complex expressions or nested logic?
- Is there any dead code or unnecessary complexity?

### 5. Best Practices
- Are language-specific best practices followed?
- Is proper error handling in place?
- Are magic numbers replaced with named constants?

### 6. Testing Considerations
- Is the code testable?
- Are there obvious edge cases to test?
- Would additional tests be beneficial?

### 7. Learning Opportunities
- What concepts could be explained to help the developer grow?
- Are there alternative approaches worth discussing?
- What patterns or techniques could improve this code?

## Instructions
Provide your review in a constructive, educational manner. For each suggestion:
1. Identify the file and location
2. Explain the issue clearly
3. Provide a learning point or explanation of why this matters
4. Suggest an improvement with an example if helpful

Rate each issue as:
- ERROR: Must fix - incorrect code or serious best practice violation
- WARNING: Should fix - improves code quality significantly
- INFO: Consider - nice-to-have improvement or learning opportunity

Remember to acknowledge good practices when you see them!
